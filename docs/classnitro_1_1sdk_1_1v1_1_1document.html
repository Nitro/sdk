Detailed Description

This class represents a handle to a PDF document.

When the last handle to a document is destroyed, all its resources are freed:

  • Each document object internally holds a file handle (if a path is specified). If a file handle is opened, it is closed.
  • If a document is unsaved, changes are lost.
  • All page objects associated with this document are invalidated.

nitro::sdk::v1::document::document ( )

Default constructor creates an empty document.

A newly created document has no pages in it.

The default constructor creates an empty document. A newly created document has no pages in it.

Think of document object as a handle to a document. Copying a document object will not create an in-memory copy of the document, you'll just have two handles to same document. Any modification done through one handle will affect the other.

◆ save() [2/2]

void nitro::sdk::v1::document::save ( std::filesystem::path const &  path)

Saves the document to the provided file-path.

The filepath is internally stored for reference. A newly created document always has to be saved with this variant first.

If this method is called on a document with an existing filepath and a different filepath is provided, it acts as save-as. The internal file reference always points to last saved filepath.
