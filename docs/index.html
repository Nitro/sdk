Introduction

This is an early, unreleased version of the Nitro SDK. It is available as-is and should not be redistributed.

Contact us for more information: https://www.gonitro.com/about/contact

API Reference

See nitro::sdk for the list of functions and classes.

Multi-threading

Unless explicitly specified, all functions and methods are safe to call from multiple threads. However, most of the functions and methods cannot operate in parallel, they will block until all preceding blocking operations are done.

Coordinate system

Most points, rectangles and sizes are relative to the PDF User Space and use PDF units.

The PDF specification defines PDF unit as 1/72nd of an inch or 72 DPI. This is also known as the printers point. For example 17pt fonts are notionally 17/72in from baseline to ascender.

The PDF User Space places the origin (0,0) at the bottom left, with the Y-axis pointing up. Correspondingly, rectangles should have their smaller coordinates in the bottom left corner.

For example: an 8 1/2" x 11" piece of paper converts to 612 PDF units (x-axis) by 792 PDF units (y-axis).

C++ Build Requirements

The supported build toolchains for linking against the nitrosdk shared library are:

  • C++17
  • On Windows, Visual Studio 2017 or 2019
  • On Linux, GNU libc 2.31 (or newer) and libstdc++ 9

Rendering

void render_document(std::filesystem::path pdf_file, uint32_t display_x, uint32_t display_y, std::filesystem::path out_root)
{
try {
auto doc = nitro::sdk::document::open(pdf_file);
auto num_pages = doc.get_num_pages();
for (nitro::sdk::page_number i = 0; i < num_pages; ++i) {
auto view = doc.get_page(i).view();
view.set_fit(nitro::sdk::fit_type::page, display_x, display_y);
view.render(out_root.string() + std::to_string(i + 1) + ".png", nitro::sdk::image_format::png);
}
}
catch (nitro::sdk::exception e) {
std::cerr << e.what() << std::endl;
throw;
}
}

Deleting Odd Pages

void remove_odd(std::filesystem::path const& path)
{
try {
auto doc = nitro::sdk::document::open(path);
nitro::sdk::page_number z = 0;
for (nitro::sdk::page_number i = 0; i < doc.get_num_pages(); i++) {
if (z++ % 2 != 1) {
doc.delete_page(i);
i--;
}
}
doc.save();
}
catch (nitro::sdk::exception const& e) {
std::cerr << e.what() << std::endl;
throw;
}
}

Batch add text to multiple files

def stamp_all_pages(in_folder, filename, out_folder):
doc = pdf.Document.open(os.path.join(in_folder, filename))
num_pages = doc.get_num_pages()
for i in range(0, num_pages):
page = doc.get_page(i)
page_rect = page.get_crop_box()
x = abs(page_rect.left - page_rect.right) * 0.2
y = abs(page_rect.top - page_rect.bottom) * 0.5
props = pdf.TextProperties()
props.font_name = "Calibri"
props.font_size = 40
page.add_text("CONFIDENTIAL", props, pdf.Point(x, y))
print(filename + ": stamped " + str(num_pages) + " pages")
doc.save(os.path.join(out_folder, filename))

print(e)
main()

Memory managment in bindings

While the C++ SDK uses RAII, per binding it tries to handle memory managment idiomatically.

C

All objects will be garbage collected. Additionally all objects extend the IDisposable interface if you wish to manually free the object.

Java

All object will be garbage collected. Additionally all objects have a deleter function 'delete_Object' that can be used to manually free the object.

Go

No object will be garbage collected. All objects must be freed manually using the provided delete function named 'DeleteObject'.

Python

All objects are garbage collected. To manually free an object, ensure there are no references left to the object and then manually run the garbage collector.
